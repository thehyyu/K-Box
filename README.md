# 💿 K-Box 卡拉OK伴唱整理系統 (Karaoke Transfer System)

> 讓爸爸媽媽告別厚重光碟！一鍵把百張伴唱帶精選轉入隨身碟，還能自動生成實體點歌本！

本系統是一個專為 Windows 本機環境設計的 **Local Web App**。它可以協助您將傳統卡拉OK伴唱光碟（VCD 的 `.dat` 與 DVD 的 `.vob`）進行**選擇性轉檔建庫**、**隨身碟循序時間戳重整**，並自動產生排版精美的 **A4 雙欄中日雙語點歌本**。

---

## ✨ 核心特色

1. **🔒 絕對安全隔離 (Sandbox Safety)**
   * **行動硬碟**：所有轉檔與資料庫操作皆限定在 `K-Box_Library/` 資料夾中，**絕對不會影響您行動硬碟中的其他重要備份或檔案**。
   * **隨身碟**：寫入歌曲與「深度重整」僅針對隨身碟的 `K-Box_Songs/` 目錄進行清空與循序重灌，**絕對不影響隨身碟內的其他無關資料**。
2. **⚡ 選擇性轉檔建庫 (Selective Ingestion)**
   * 放入光碟後，父母只需**勾選**自己想唱的歌曲進行轉檔，不需要浪費大量時間與硬碟空間去轉碼整張 CD 不喜歡的歌。
3. **👴 長輩免打字設計 (Elderly Friendly)**
   * 系統會自動為每張光碟生成時間戳記編號（例如 `CD_20260616_1550`），父母建庫時**完全不需要命名光碟**。
   * 轉檔時如果嫌打字太累，**歌名可直接留空**。系統會以 `Track XX` 作為預設檔名直接轉檔，標記為「待補檔」，事後再由子女（您）一併補打。
4. **🎞️ DVD 章節自動分割 (DVD Autocut)**
   * 支援將單一大型 DVD 影音軌透過 `ffprobe` 解析章節，並使用 `ffmpeg` 自動切割為單首獨立的 MP4 歌曲檔案。
5. **📺 舊型 DVD 播放器硬體排序相容 (Sequential Write & UTime Touch)**
   * 針對舊型播放器只看隨身碟寫入時間的限制，系統匯出時採用 **「單執行緒循序寫入」** 並透過 Python 的 `os.utime()` 強制將檔案修改時間修改為**等差遞增**，保證播放器點歌順序 100% 正確。
6. **🇯🇵 日語漢字與平/片假名完美相容**
   * 資料庫與點歌本採用全 UTF-8 編碼與中日共用系統字型，日文歌曲與歌手名稱顯示清晰、列印時絕對不缺字。

## 📂 專案目錄結構

```
K-Box/
├── backend/
│   ├── bin/                # 【重要】放置 ffmpeg.exe 與 ffprobe.exe
│   ├── config.py           # 行動硬碟路徑、FFmpeg 路徑與安全目錄定義
│   ├── database.py         # library.json 的 CRUD 執行器
│   ├── scanner.py          # Windows 光碟機掃描、DAT/VOB 偵測、DVD 章節解析
│   ├── converter.py        # 背景 FFmpeg 轉檔 queue 運作、即時進度解析
│   └── exporter.py         # USB 偵測、循序寫入與時間戳重置、HTML 點歌本生成
├── frontend/
│   ├── index.html          # SPA 主網頁
│   ├── style.css           # 專屬 CSS（大字體、高對比、玻璃擬物化風格）
│   └── app.js              # Vanilla JS 狀態管理器與 API 呼叫器
├── tests/                  # 16 個自動化 Unit / Integration 測試
├── requirements.txt        # python 相容套件
├── start_kbox.sh           # macOS 快速啟動腳本 (測試用)
└── 啟動K-Box.bat            # Windows 一鍵啟動腳本 (爸媽用)
```

## 🛠️ 安裝與建置指南 (子女設定步驟)

在交給父母使用前，請先在他們的 Windows 電腦上進行一次性設定：

### 步驟 1：安裝 Python
* 請至 [Python 官方網站](https://www.python.org/) 下載並安裝 Python 3.10 或更新版本（安裝時請務必勾選 **"Add Python to PATH"**）。

### 步驟 2：下載並解壓專案
* 將本專案（`K-Box`）下載或 Git Clone 到電腦中。

### 步驟 3：安裝依賴套件 (venv)
在專案根目錄下打開命令提示字元 (cmd) 並執行：
```cmd
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
```

### 步驟 4：置放 FFmpeg 執行檔 (關鍵)
1. 前往 [FFmpeg 官網](https://ffmpeg.org/download.html) 或使用第三方下載 Windows 64-bit 版本的 Static Binaries。
2. 解壓後，將 **`ffmpeg.exe`** 與 **`ffprobe.exe`** 兩個檔案複製並放置到專案的 **`backend/bin/`** 資料夾中。
   *(如此一來，程式會優先使用此目錄的執行檔，不需去設定複雜的 Windows 環境變數)*。

## 🚀 系統啟動方式

* **Windows (爸媽使用)**：直接點兩下執行根目錄底下的 **`啟動K-Box.bat`**。
* **macOS (子女測試)**：在終端機中執行 `./start_kbox.sh`。

啟動後，系統會自動在背景跑起後端，並呼叫您的預設瀏覽器開啟 `http://localhost:8080/` 點歌控制台。

## 📖 使用者操作指南 (User Guide)

### 【第一階段：放入光碟選歌建庫】(適合爸媽操作)

1. **放入光碟**：將伴唱光碟放入電腦光碟機中。
2. **掃描光碟**：在「**光碟選歌建庫**」頁面中，點擊下拉選單選擇您的光碟機代號（例如 `D:\`），然後點選「**🔍 掃描光碟**」。
3. **勾選精選**：畫面會列出光碟內所有的歌曲音軌。**僅勾選**您想要唱的歌。
4. **輸入歌名 (可跳過)**：
   * 在勾選的歌曲後方，對照實體光碟殼上的曲目，輸入：`歌名 - 歌手`（例如：`川の流れのように - 美空ひばり`）。
   * **若嫌打字太累，可直接留空不打字**，直接進行下一步。
5. **開始轉檔**：點擊最下方綠色的「**🚀 開始轉檔已選歌曲**」大按鈕。
6. **取出換片**：轉檔進度條跑完並發出提示音後，即可取出光碟，放入下一張繼續重複上述步驟。

---

### 【補充：子女後續補檔與整理】(適合您有空時操作)

1. 打開網頁，點選左側「**精選曲庫整理**」。
2. 點擊「**⚠️ 待補檔歌曲**」按鈕，系統會自動篩選出所有先前爸媽留空未打字的歌曲。
3. 拿著實體光碟外殼，對照匯入的時間戳專輯，點擊歌曲右側的「**✏️ 編輯**」按鈕。
4. 輸入正確的歌名與歌手後點擊「**儲存**」，歌曲便會轉為「綠色已完成」狀態，專輯也會自動消除待補檔警告。

---

### 【第二階段：隨身碟同步與列印點歌本】(適合爸媽或您操作)

1. **插上隨身碟**：將要帶去卡拉OK店的 USB 隨身碟插上電腦。
2. **選擇隨身碟**：在網頁點選「**隨身碟同步匯出**」，於下拉選單選擇該隨身碟路徑。
3. **選擇歌曲**：在右側表格中，勾選本次想帶出門的歌曲（預設會自動全選曲庫內的所有精選歌曲）。
4. **選擇同步策略**：
   * **差額同步 (預設)**：只複製隨身碟裡還沒有的新歌，速度最快。
   * **深度重整 (推薦)**：當隨身碟內歌曲順序亂掉時，勾選此項。系統會自動**清空隨身碟內的 `K-Box_Songs` 資料夾，並重新按照 KTV 編號順序一首接一首重新複製**，保證 DVD 播放器排序 100% 正確。
5. **開始寫入**：點擊「**🚀 開始同步到隨身碟**」，等待進度條完成。
6. **列印歌單**：點擊「**🖨️ 開啟列印歌本**」，會開啟一個排版精美的 A4 雙欄歌單網頁。按下鍵盤的 **`Ctrl + P`**，即可直接用印表機印出，或是儲存為 PDF 檔案隨身攜帶！

## ❓ 常見問題與排除 (Troubleshooting)

#### Q1: 點擊「掃描光碟」後，系統提示「找不到 FFmpeg 執行檔」？
* **排除方法**：請確認您是否已依照安裝指南，將 `ffmpeg.exe` 與 `ffprobe.exe` 放置於 `backend/bin/` 資料夾內。

#### Q2: 隨身碟插在卡拉OK機上播放時，歌曲順序與紙本點歌本的編號對不起來？
* **排除方法**：這是因為卡拉OK播放器只看實體硬碟寫入順序。請在匯出隨身碟時，勾選「**深度重整**」選項，讓系統將隨身碟專屬目錄清空後重新循序寫入一次即可解決。

#### Q3: 日文歌曲列印出來會顯示成亂碼或豆腐字嗎？
* **排除方法**：本系統採用 Noto Sans TC / 微軟正黑體等萬國碼字型，並強制使用 UTF-8。不論在網頁上還是在列印點歌本中，日文平假名、片假名與日文漢字皆能完美呈現。
